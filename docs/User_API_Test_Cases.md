## API 测试用例

### 1. 用户注册

**POST** `/api/auth/users/register`

#### 测试用例 1.1: 成功注册
```json
{
  "username": "testuser1",
  "password": "password123",
  "phone": "13800138999",
  "name":"测试用户"
}
```

**期望响应**:
```json
{
    "code": 200,
    "data": {
        "phone": "13800138999",
        "userId": 10,
        "username": "testuser1"
    },
    "message": "success"
}
```

#### 测试用例 1.2: 用户名已存在（应失败）
```json
{
  "username": "testuser1",
  "password": "password123",
  "phone": "13800138999"
}
```

#### 测试用例 1.3: 缺少必填字段（应失败）
```json
{
  "username": "incomplete",
  "password": ""
}
```

## 测试注意事项

1. **测试顺序**: 建议按照用户注册 → 登录 → 添加车辆 → 提交维修请求的顺序进行测试
2. **数据依赖**: 某些测试用例依赖于前面创建的数据，请注意测试顺序
3. **错误场景**: 每个API都包含了正常和异常场景的测试用例
4. **数据清理**: 测试完成后建议清理测试数据，避免影响后续测试
